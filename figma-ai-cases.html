案例一：用 API 代码自动生成登录页面

不再"生成一张图片"，而是执行结构化代码创建可编辑图层。AI 通过 MCP Server 调用 Figma 原生 API → 创建画布帧 → 放置 Logo → 添加输入框组件 → 创建按钮 → 启用 Auto Layout。

login-screen.ts — AI Agent 执行的绘图脚本
import { useFigma } from "@figma/ai-sdk";

async function generateLoginScreen() {
  const figma = useFigma({
    token: process.env.FIGMA_TOKEN,
    fileKey: "YOUR_FILE_KEY",
  });
  // ① 创建主画布 (iPhone尺寸)
  const frame = await figma.createFrame({ name:"Login Screen", width:375, height:667 });
  // ② 放置品牌 Logo
  await figma.placeAsset({ parentId:frame.id, assetUrl:"https://brand.com/logo.png", width:120, height:120 });
  // ③ 邮箱输入框
  await figma.createComponent({ parentId:frame.id, name:"Email Input", type:"text_input", width:335, height:48, placeholder:"Enter email" });
  // ④ 密码输入框（安全模式）
  await figma.createComponent({ parentId:frame.id, name:"Password Input", type:"text_input", width:335, height:48, placeholder:"Enter password", isSecure:true });
  // ⑤ 登录按钮（iOS蓝）
  await figma.createComponent({ parentId:frame.id, name:"Login Button", type:"button", width:335, height:48, label:"Log In", backgroundColor:"#007AFF" });
  // ⑥ 启用垂直 Auto Layout（响应式）
  await figma.createLayout({ targetId:frame.id, mode:"vertical", spacing:20, padding:20 });
  return frame;
}

✨ 输出结果

AI 在 Figma 中生成完整 iPhone 登录页：
• 所有元素都是可编辑的独立图层（非图片）
• Auto Layout 已启用，支持响应式调整
• 组件已命名，开发者可直接引用
• 整个过程无需人工操作画布

可编辑图层Auto Layout组件化~30秒完成
📝 Skills Markdown

案例二：用 Markdown 定义按钮组件系统（36变体自动生成）

Figma Skills 是 Markdown 格式的「设计食谱」。你告诉 AI "怎么设计"，它严格按规范在画布上执行。以下定义 3样式×3尺寸×4状态 = 36个按钮变体：

button-system.skill.md — AI 读取并执行的「设计食谱」
# Button Component System (按钮组件系统)

## Objective
Create a complete button component library with variants.

## Variants Required
- Styles: Primary, Secondary, Danger
- Sizes: Small (32px), Medium (48px), Large (56px)
- States: Default, Hover, Active, Disabled

## Step 1: Create Main Component
1. Frame named "Button/Primary/Medium" · 200×48px
2. Fill: #007AFF · Border radius: 8px · Text: size 16 Bold

## Step 2: Generate Variants
For each style × size: auto-generate frame with consistent spacing

## Step 3: Add Interactive States
Default: opacity 100% | Hover: −10% brightness
Active: −20% brightness | Disabled: opacity 50%

## Step 4: Create Design Tokens
color-primary: #007AFF | color-secondary: "#E5E5EA | spacing: 8/16/24px

## Safety Constraints
- ❌ Never delete existing components
- ✅ Maintain 8px grid alignment
- ✅ Ensure WCAG AA contrast minimum

✨ AI 执行结果

AI 读取这份 Markdown 后，在 Figma 中自动生成 36 个按钮变体： Primary / Secondary / Danger × S / M / L × Default / Hover / Active / Disabled。 全部遵循 8px 网格对齐 + WCAG AA 对比度标准，同时输出对应的设计令牌（Design Tokens），零人工干预。

💻 Code Connect + MCP

案例三：Code Connect — Figma 组件 ↔ React 代码双向同步

当 AI 在画布上修改组件后，如何保证代码与设计完全一致？通过 Code Connect 配置文件将 Figma 组件属性映射为代码 Props。画布改了，代码跟着改。

figma.config.ts — Code Connect 映射配置
import { figma } from "@figma/code-connect";

// 将 Figma 组件映射到 React 代码组件
figma.connect(
  "Button/Primary/Medium",     // ← Figma 组件路径
  {
    component: "./components/Button.tsx",
    props: {
      variant:  figma.enum("Variant"),       // Primary|Secondary|Danger
      size:     figma.enum("Size"),          // S|M|L
      disabled: figma.boolean("Disabled"),
      label:    figma.string("Label"),
    },
  },
);
// 使用效果：AI在Figma中改为Danger+Disabled → React代码自动变为:
// <Button variant="danger" disabled>...</Button>

🔄 双向同步工作流

① AI Agent 在 Figma 画布修改按钮（改色/换尺寸/加状态）
↓  ② Code Connect 通过 MCP Server 自动读取属性变更
↓  ③ React/Swift/Kotlin 代码自动更新（Props与设计一致）
↓  ④ 开发者直接使用最新代码，无需对照设计稿

实时双向同步React/Swift/KotlinProps自动映射

⚠️ 当前局限（实测结论）
  • Make 生成的代码不具备生产级质量—— 缺乏无障碍性、语义化标签、模块化结构，需工程师重构
  • First Draft 无法使用你的设计系统—— 仅基于 Figma 自有 UI 库，不支持企业自定义组件库
  • 不常见设计模式无法识别—— 创新性布局或特殊交互需要人工设计
  • 不会创建原型连线—— First Draft 出来的屏幕之间没有跳转关系
  • 视觉细腻度不够—— AI 生成的设计有明显的「AI 生成感」，缺乏专业细节打磨
  • Make 定位是预览工具—— 可发布自定义域名但不包含 SEO，不适合生产级网站
  • 需 Pro 或以上账户—— AI Agent 画布功能需要付费订阅才能使用
💡 推荐使用路径
1
使用 First Draft 快速出初稿（文字描述 → UI线框图/高保真）
2
用 Make an Image 补充产品配图和视觉元素
3
进入 Dev Mode 获取前端代码框架（注意：需人工重构优化）
4
团队使用 Skills 系统 定义设计规范，确保AI输出一致性
5
开发者通过 MCP Server 在 IDE 中直接调用 Figma 画布能力
